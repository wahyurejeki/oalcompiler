# OAL Compiler for Laravel

This project is an implementation of a compiler for **OAL (Object Action Language)** integrated into the **Laravel** PHP framework. The process of parsing OAL source code is done using **ANTLR v4** to generate a parser from a defined grammar.

## Description

This compiler is designed to parse OAL source code, perform syntactic and semantic analysis, and (eventually) produce an intermediate representation of the code. By integrating it into Laravel, we can leverage modern PHP features and the Laravel ecosystem, such as Artisan commands for execution, and ease of project structure management.

## Technologies Used

*   **Laravel**: The main application framework.
*   **PHP 8.1** (or newer).
*   **ANTLR v4**: Parser generator.
*   **Composer**: PHP dependency management.

## Installation and Setup

Follow these steps to run this project in your local environment.

1.  **Clone the Repository**
    ```bash
    git clone https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
    cd YOUR_REPO_NAME
    ```
    *(Replace `YOUR_USERNAME` and `YOUR_REPO_NAME` with your repository details)*

2.  **Install PHP Dependencies**
    Make sure you have Composer installed, then run:
    ```bash
    composer install
    ```

3.  **Configure Laravel Environment**
    Copy the environment file and generate the application key:
    ```bash
    cp .env.example .env
    php artisan key:generate
    ```
    *(Adjust the `.env` file if necessary, for example for database connections)*

4.  **Generate ANTLR Parser**
    Make sure you have ANTLR v4 installed on your system. If you make changes to the `OAL.g4` grammar file, you need to regenerate the PHP parser files. Use the following command from the project root directory:

    ```bash
    antlr4 -Dlanguage=PHP -visitor OAL.g4 -o src/OalLib
    ```
    This command will generate the PHP parser classes from `OAL.g4` into the `src/OalLib` directory, complete with the *Visitor Pattern* implementation (`-visitor`).

## Usage

The most common way to run this compiler is through a custom Artisan command.

**Example Command:**
```bash
php artisan oal:compile "examples/program.oal"
```
*(Assuming you have created an `oal:compile` command)*

This command will take the path to the `.oal` file as an argument, process it using the generated parser, and display the results, such as the Abstract Syntax Tree (AST), error messages, or other compilation output.

## Project Structure

*   `OAL.g4`: The ANTLR grammar file that defines the OAL language syntax.
*   `src/OalLib/`: The output directory for the parser, lexer, visitor, and listener classes generated by ANTLR.
*   `app/Console/Commands/`: The ideal location to place the Artisan command logic (e.g., `OalCompileCommand.php`).
*   `examples/`: A directory containing example code in the OAL language for testing.

## Visual Editor Studio

We have built a premium, web-based visual editor dashboard directly at the root directory that allows you to visually design your Class Diagram (models, attributes, relationships), map routes, configure middleware guards, and compile them to Laravel with a single click.

### How to Run the Visual Editor:

1. Start PHP's built-in local development server from the project root:
   ```bash
   php -S localhost:8000
   ```
2. Open your web browser and navigate to:
   ```text
   http://localhost:8000/
   ```

### Key Features:
* **Interactive Canvas**: Drag and drop models, define field names, set data types (`string`, `integer`, `boolean`, etc.), and toggle modifiers (`primary`, `unique`, `nullable`, `default`).
* **Visual Relationships**: Connect models visually. Simply click the link anchor of Model A and target Model B to define `hasMany`, `belongsTo`, `hasOne`, or `belongsToMany` relationships.
* **Route Manager**: Add/edit routes visually using a dynamic table, mapping HTTP methods and paths to controller actions, with easy middleware checkbox guards.
* **Middleware Wizard**: Add middlewares using templates like API Token Check, Session Check, or Custom OAL logic.
* **Real-time Code Preview**: View the generated OAL code in real-time as you build your diagrams.
* **One-click Compiler**: Click "Compile to Laravel" to automatically generate the `.oal` code and run the ANTLR PHP compiler (`generate.php`) right from your browser, showing full compiler console output.
